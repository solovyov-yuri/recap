# UX-моки и дизайн-направление

## 0. Моки

Главный визуальный источник для реализации: `docs/ui-reference/index.html`.

Он согласует размеры, spacing, цвета и состояния между экранами. Если HTML-референс и PNG-моки расходятся, агент должен следовать HTML-референсу.

Вторичные визуальные ориентиры для MVP:

- `docs/mockups/01-main-running.png` - выбран файл, идет транскрибация, видны прогресс и live-log.
- `docs/mockups/02-result-partial-success.png` - transcript сохранен, summary упал, показан partial success recovery.
- `docs/mockups/03-settings.png` - экран настроек с секциями и keychain-состоянием API key.
- `docs/mockups/icons/recap-icon-dimensional-1.png` - выбранное направление `dimensional 1`, чистовая иконка.
- `docs/mockups/icons/recap-icon-dimensional-1-preview.png` - проверка выбранной иконки в размерах 32/64/128/256px.

## 1. Общий стиль

Интерфейс Recap desktop должен ощущаться как легкий рабочий инструмент, а не как веб-лендинг.

Ориентиры:

- Linear;
- Raycast;
- Arc settings;
- аккуратные native desktop utilities.

Визуальные правила:

- светлая тема по умолчанию;
- темная тема желательна, но не обязательна для первого технического slice;
- фон приложения: мягкий off-white;
- панели белые или почти белые;
- тонкие серые border-разделители;
- радиусы 6-8px;
- restrained cobalt или teal accent;
- не использовать фиолетовые градиенты, декоративные blobs/orbs, маркетинговые hero-блоки;
- интерфейс должен быть плотным, но не перегруженным.

## 2. Главный экран

Композиция:

```text
+------------------+---------------------------------------------+----------------------+
| Sidebar          | Main workspace                              | Inspector            |
|                  |                                             |                      |
| Recap            | meeting_2026_06_19.mp3                      | Настройки запуска    |
| Новый разбор     | 01:24:12 · 78 MB                            |                      |
| История          |                                             | Provider             |
| Настройки        | [Запустить]                                 | Model                |
|                  |                                             | Language             |
| Последние        | Подготовка       done                       | Mode                 |
| ...              | Транскрибация    running 42%                | Chunking             |
|                  | Суммаризация     waiting                    | Output               |
|                  | Экспорт          waiting                    |                      |
|                  |                                             |                      |
|                  | [Транскрипт] [Резюме] [Лог]                 |                      |
|                  |                                             |                      |
+------------------+---------------------------------------------+----------------------+
```

Левая sidebar:

- `Recap`;
- `Новый разбор`;
- `История`;
- `Настройки`;
- compact recent jobs list.

Центр:

- drop zone до выбора файла;
- после выбора показывает имя файла и базовые метаданные;
- button `Запустить`;
- progress steps;
- tabs результата.

Правая панель:

- настройки текущего запуска;
- компактные selects/toggles/inputs;
- без API key, ключи живут на Settings screen.

## 3. Состояние запуска

Во время выполнения:

- кнопка `Запустить` становится disabled;
- появляется кнопка `Остановить`, если bridge поддерживает cancel;
- этапы показывают статус:
  - `Готово`;
  - `Выполняется`;
  - `Ожидает`;
  - `Ошибка`.
- log tab обновляется live.

Пример этапов:

```text
Подготовка       Готово
Транскрибация    42%
Суммаризация     Ожидает
Экспорт          Ожидает
```

Live-log должен быть user-facing:

```text
Файл выбран: meeting.mp3
Подготовка аудио включена
Загружаю модель faster-whisper: large-v3
Транскрибация началась
Транскрипт сохранен: C:\...\transcript.txt
Суммаризация началась: ollama / qwen3.5:latest
```

## 4. Result view

Tabs:

- `Транскрипт`;
- `Резюме`;
- `Лог`.

Transcript:

- readonly;
- timestamp слева;
- текст сегмента справа;
- удобное копирование всего transcript можно добавить, но не обязательно.

Summary:

- editable;
- форматированный preview не обязателен, достаточно хорошего textarea/editor;
- основные actions закреплены сверху или снизу.

Actions:

- `Копировать`;
- `Экспорт`;
- `Открыть папку`;
- `Повторить`.

При partial success:

```text
Транскрипт сохранен, но резюме не создано.
Причина: LLM timeout.

[Повторить суммаризацию] [Открыть транскрипт] [Настройки]
```

## 5. Settings screen

Settings не должен быть одним бесконечным полотном. Лучше layout:

```text
+----------------------+---------------------------------------+
| Settings nav         | Section content                       |
|                      |                                       |
| Транскрибация        | Model                                |
| Суммаризация         | Device                               |
| Предобработка        | Compute type                         |
| Пути                 | ...                                  |
| Приватность          |                                       |
| Ключи API            |                                       |
+----------------------+---------------------------------------+
```

Секции:

- `Транскрибация`;
- `Суммаризация`;
- `Предобработка`;
- `Пути`;
- `Приватность`;
- `Ключи API`.

Footer:

- `Сохранить`;
- `Отменить`;
- dirty state indicator.

## 6. История

History list item:

```text
meeting_2026_06_19.mp3
Сегодня, 12:04 · Успешно · ollama / medium
```

Actions:

- открыть;
- открыть папку;
- повторить;
- удалить из истории.

История не должна удалять исходные файлы или результаты без отдельного явного подтверждения.

## 7. Empty states

Первый запуск:

- не делать marketing hero;
- показать compact drop zone;
- справа показать текущие настройки;
- снизу можно показать пустую историю.

Текст:

```text
Выберите аудиофайл встречи
```

Подсказка:

```text
Поддерживаются WAV, MP3, M4A, OGG.
```

## 8. Micro-interactions

- drag-and-drop highlight;
- disabled states на кнопках во время run;
- toast после копирования;
- toast после экспорта;
- inline validation для настроек;
- warning pill при внешнем provider без `privacy_ack`;
- masked API key;
- command buttons с lucide icons.

## 9. Цвета и типографика

Пример палитры:

```text
App background: #F6F7F8
Panel:          #FFFFFF
Border:         #E3E6EA
Text primary:   #171A1F
Text muted:     #6B7280
Accent:         #2563EB или #0F766E
Success:        #15803D
Warning:        #B45309
Error:          #B91C1C
```

Не использовать интерфейс, построенный почти целиком на одной цветовой семье.
